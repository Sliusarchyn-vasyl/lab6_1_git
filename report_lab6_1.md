## Опис виконання лабораторної роботи

У цій лабораторній роботі я налаштував локальний репозиторій Git, пов'язав його з GitHub,
додав файл лабораторної роботи №3 у форматі PDF та виконав базові операції push/pull.


У межах лабораторної роботи я додатково попрацював з онлайн-тренажером Git, де послідовно виконав три розділи: «Вступ», «Їдемо далі» та «Переміщуємо роботу туди-сюди». Тренажер дає змогу наочно побачити, як змінюється історія репозиторію під час виконання різних команд Git.

У розділі «Вступ» я закріпив базові дії: створення комітів, перегляд історії та розуміння положення покажчика HEAD. Для цього використовувалися команди на зразок git commit, git log, git branch, git checkout. Після виконання всіх кроків тренажер підтвердив успішне проходження розділу.

У розділі «Їдемо далі» основна увага приділялась роботі з гілками. Я створював нові гілки для окремих змін, перемикався між ними та об’єднував їх з основною гілкою за допомогою git merge. Це допомогло краще зрозуміти, як організовується паралельна робота над проєктом і як зміни з різних гілок потрапляють до спільної історії.

У розділі «Переміщуємо роботу туди-сюди» я тренувався керувати історією репозиторію: повертати стан проєкту на попередні коміти, скасовувати зміни та переносити окремі коміти в інші гілки. Для цього застосовувалися команди git reset, git revert та git cherry-pick. Завдяки цьому я отримав уявлення про те, як виправляти помилки в історії, акуратно скасовувати коміти та переносити потрібні зміни без повного переписування проєкту.

Підсумок по тренажеру: проходження цих трьох розділів дозволило не тільки теоретично, а й практично закріпити роботу з гілками, комітами та історією в Git. Отримані навички можна застосовувати при роботі з власними та командними репозиторіями на GitHub.
